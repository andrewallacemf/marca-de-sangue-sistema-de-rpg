# 👋 Comece aqui

Bem-vindo(a). Se você nunca usou GitHub na vida, **este é o lugar certo para começar**.
Leia com calma. Não precisa entender tudo de uma vez.

---

## O que é este projeto, em linguagem humana

Imagine uma **pasta compartilhada na nuvem**, cheia de documentos de texto sobre nosso
RPG. Só que essa pasta tem três superpoderes que uma pasta comum não tem:

1. **Histórico infinito.** Toda alteração fica guardada com o nome de quem fez e quando.
   Se alguém apagar algo por engano, dá pra voltar atrás.
2. **Sincronização controlada.** Você trabalha numa cópia no *seu* computador e, quando
   está pronto(a), "envia" suas mudanças para a versão oficial na nuvem.
3. **Pode ser editada por IA.** Você pode pedir para uma inteligência artificial escrever,
   corrigir ou reorganizar o conteúdo — e as mudanças entram no mesmo lugar.

Essa "pasta com superpoderes" se chama **repositório**. Ele mora no **GitHub** (a nuvem).
A ferramenta que faz a mágica de sincronizar se chama **Git**. Você quase não vai precisar
pensar nesses nomes — a gente usa um aplicativo visual que faz o trabalho pesado.

## As 5 palavras que você vai ouvir sempre

Não decore. Só saiba que existem. O [glossário](manuais/glossario.md) explica todas.

| Palavra | Tradução mental |
|---------|-----------------|
| **Repositório** (repo) | A pasta oficial do projeto na nuvem. |
| **Clonar** | Baixar uma cópia da pasta para o seu computador (você faz **uma vez só**). |
| **Pull** (puxar) | Baixar as novidades que os outros fizeram. Faça **sempre que for começar**. |
| **Commit** (carimbar) | Registrar suas mudanças com uma frase explicando o que fez. |
| **Push** (subir) | Enviar seus commits para a nuvem, para os outros verem. Faça **quando terminar**. |

## A regra de ouro (se você só lembrar de uma coisa, lembre desta)

> ### 🟢 **PUXE no começo. SUBA no fim.**
> Sempre dê **pull** antes de começar a trabalhar e **push** quando terminar.
> 90% dos problemas somem quando todo mundo faz isso.
