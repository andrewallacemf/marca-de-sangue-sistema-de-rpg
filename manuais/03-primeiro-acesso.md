# 🔑 03 — Primeiro acesso

*Você faz isto **uma vez** por computador: baixar (clonar) o projeto.*

Depois de aceitar o convite por e-mail e instalar o GitHub Desktop, é hora de trazer o
projeto para o seu computador.

---

## Passo a passo (pelo GitHub Desktop)

1. Abra o **GitHub Desktop** (já logado na sua conta).
2. No menu superior, clique em **File → Clone repository**
   (ou no botão **Clone a repository from the Internet**).
3. Na aba **GitHub.com**, você verá a lista de projetos aos quais tem acesso. Encontre o
   repositório do RPG (o dono te diz o nome exato).
4. Em **Local path**, escolha **onde** no seu computador a pasta vai ficar.
   Sugestão: uma pasta fácil de achar, como `Documentos/RPG`.
5. Clique em **Clone**.

Pronto. O GitHub Desktop baixou uma **cópia completa** do projeto para aquela pasta.
Essa cópia é **sua** — você trabalha nela à vontade, e só o que você "enviar" chega aos outros.

> **Clonar ≠ baixar um zip.** Ao clonar, sua pasta fica "ligada" à nuvem. É isso que
> permite puxar novidades e enviar mudanças depois. Nunca baixe o projeto como `.zip`
> pelo site para trabalhar — isso quebra a ligação.

## Onde ficam os arquivos

Abra a pasta que você escolheu (ex.: `Documentos/RPG`). Você verá as pastas
`sistema-base/`, `cenarios/`, `manuais/` etc. — exatamente como aparecem no GitHub. São
**arquivos de verdade** no seu computador. Você pode abri-los no Obsidian, no VS Code ou
em qualquer editor.

## Abrindo no Obsidian (opcional, recomendado)

1. Abra o Obsidian → **Open folder as vault**.
2. Aponte para a pasta que você clonou.
3. Comece pelo arquivo `README.md`.

## Conferindo que deu certo

No GitHub Desktop, o topo deve mostrar o nome do repositório e a mensagem
**"No local changes"** (nenhuma mudança local) — sinal de que sua cópia está limpa e
igual à nuvem. É desse estado que você sempre parte.

---

## ⚠️ Só faça isto uma vez

Clonar é só na primeira vez, **por computador**. Depois disso, você **nunca mais clona** —
você **puxa** (pull) as novidades, o que é diferente e muito mais rápido. Isso está no
próximo manual, que é o mais importante de todos.
