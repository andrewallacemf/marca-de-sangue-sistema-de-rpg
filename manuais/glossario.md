# 📚 Glossário

*Todos os termos que você vai encontrar, explicados em linguagem de gente.*

**Repositório (repo)**
A pasta oficial do projeto. Existe na nuvem (no GitHub) e em cópias no computador de cada
pessoa.

**GitHub**
O site/nuvem onde mora a versão oficial do repositório. É lá que todo mundo se encontra.

**Git**
O programa que sincroniza sua cópia com a nuvem (baixa novidades, registra e envia
mudanças). Trabalha nos bastidores.

**GitHub Desktop**
O aplicativo visual com botões que comanda o Git para você, sem linha de comando. Sua
ferramenta principal do dia a dia.

**Clonar**
Baixar, pela primeira vez, uma cópia completa do repositório para o seu computador. Você
faz uma vez por computador.

**Cópia local**
A versão do projeto que está no *seu* computador, onde você trabalha. As mudanças só
chegam aos outros quando você faz push.

**Pull (puxar)**
Baixar para a sua cópia as novidades que outras pessoas já enviaram. Faça sempre antes de
começar.

**Fetch**
Uma espiada na nuvem para ver se há novidades, **sem** baixá-las ainda. O GitHub Desktop
faz isso quando você clica em "Fetch origin".

**Commit (carimbar)**
Registrar um conjunto de mudanças com uma frase que explica o que você fez. É um "ponto de
salvamento" com etiqueta.

**Mensagem de commit**
A frase curta que descreve o commit (ex.: `adiciona magia de cura`). Vira parte do
histórico.

**Push (subir/enviar)**
Enviar seus commits da sua cópia para a nuvem, para que os outros possam vê-los. Faça no
final.

**Origin**
Apelido para "a versão na nuvem". Quando o app diz "Push origin", quer dizer "enviar para a
nuvem".

**Main**
O nome da linha principal do projeto — a versão oficial onde todo mundo trabalha.

**Conflito**
Quando duas pessoas mudaram o mesmo trecho do mesmo arquivo e o sistema pede que você
escolha o texto final. Tem [manual próprio](06-conflitos.md).

**Merge (juntar)**
O ato de combinar mudanças de diferentes pessoas. Na maioria das vezes acontece
automaticamente; às vezes gera um conflito para você resolver.

**Changes (mudanças)**
A aba do GitHub Desktop que mostra tudo que você alterou desde o último commit.

**History (histórico)**
A aba do GitHub Desktop que mostra a lista de todos os commits já feitos, por todo mundo.

**Markdown (.md)**
O formato de texto simples em que tudo é escrito. Usa símbolos leves como `#` (título),
`-` (lista) e `**negrito**`.

**Frontmatter**
O pequeno bloco entre `---` no começo de cada arquivo de conteúdo, com informações como
título, tipo e status. Ver [CONVENCOES.md](../CONVENCOES.md).

**Template (modelo)**
Um arquivo-molde (nome começa com `_`) que você **copia** para criar conteúdo novo no
formato certo. Nunca escreva dentro do template original.

**.gitignore**
Uma lista de arquivos que o projeto deve **ignorar** (lixo de sistema, caches). Assim eles
não entram no repositório.

**Guardião do Git**
A pessoa da equipe combinada para ajudar a destravar casos difíceis (conflitos grandes,
recuperar algo). Chamar não é vergonha, é o certo.
